The challenge

Generic AI Doesn't Know Your Environment

What AI knows

  • Public OpenShift documentation
  • Community forum answers
  • Generic best practices
  • Standard YAML manifests

What AI doesn't know

  • Your internal runbooks
  • Environment-specific configs
  • Compliance & security policies
  • Custom operational procedures
It's Not Just Data

It's your data — internal runbooks, compliance policies, environment configs, and operational expertise that sit outside any public knowledge base.

The solution

Bring Your Own Knowledge Bridges the Gap

Augment Lightspeed with your organization's private documentation using Retrieval-Augmented Generation. No model fine-tuning required — your data stays in your environment.

Your Docs

Markdown files containing runbooks, policies, and configurations

RAG Engine

Embeddings search finds relevant context at query time

Tailored Answers

LLM responds with your organization's context and policies

Building the knowledge image

Build and Push Your Knowledge

# Run the RAG tool to build embeddings from your docs
$ podman run -it --rm --device=/dev/fuse \
-v $XDG_RUNTIME_DIR/containers/auth.json:/run/user/0/containers/auth.json:Z \
-v /path/to/content:/markdown:Z \
-v /my/output/path:/output:Z \
registry.redhat.io/openshift-lightspeed-tech-preview/lightspeed-rag-tool-rhel9:latest
# Load, tag, and push the resulting image
$ podman load -i /my/output/path/byok-image.tar
$ podman tag localhost/byok-image:latest <registry>/openshift-lightspeed/myimage-byok:latest
$ podman push <registry>/openshift-lightspeed/myimage-byok:latest
Configuration

Point Lightspeed at Your Knowledge

apiVersion: ols.openshift.io/v1alpha1
kind: OLSConfig
spec:
llm:
providers:
- name: myProvider
type: openai
credentialsSecretRef:
name: openai-api-keys
models:
- name: gpt-4o
ols:
defaultModel: gpt-4o
defaultProvider: myProvider
rag: # Add this stanza
- image: <registry>/openshift-lightspeed/myimage-byok:latest
indexID: vector_db_index
indexPath: /rag/vector_db
